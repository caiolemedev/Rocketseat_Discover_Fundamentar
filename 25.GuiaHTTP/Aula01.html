HTTP - fluxo de menssagem em usuário e servidor.

Visão Geral

HTTP - HyperText Transfer Protocol, um hypertexto pode conter audios, links, videos. O HTTP permite a troca de dados e informações na internet e abre uma chamada para cada um dos recursos.

Visualizando a comunicação

É basicamente uma troca de menssagens entre Browser e Server enviando e recebendo e menssagens. No pedido, temos os methods (define a ação e o tipo do pedido, GET ou POST), os headers e o body (similar aos de response). Na resposta temos o Status Code (tipo de resposta, 404, 201, 500, etc.), o headers (campos propriedade-valor, ex: URL, user agent, etc.) e o body (conteúdo da menssagem, ex: HTML, JSON, etc.).

Visualizando com DevTools

Usando o DevTools e acessando a aba network podemos visualizar todos os requests e responses das páginas que estamos acessando. Podemos visualizar todos os textos e todos os sub-pedidos.

Visualizando com cURL

Ferramenta já está instalada nos sistemas unix (Linux e Mac), no Windows podemos usar o gitbash. Usamos essas ferramentas pois para protocólos seguros como o HTTPS não conseguimos ver todos os detalhes nas ferramentas de desenvolvedor. Podemos usar o -i e -v antes do site para recebermos mais detalhes da menssagem.

Tags de Entrada de Dados

Input

Mais usada em formulários e possui várias combinações pelo atributo type. Mas precisa de um name e um id também. Tipos: text, file, password, etc...

Atributos do input

Type aceita muitos tipos. Autocomplete permite a sugestão pelo navegador. Autofocus fará o cursor ficar neste campo ao iniciar a página. Disabled bloqueia o preenchimento do formulário e deixa opaco. Readonly permite a leitura (sem opaco) mas não permite edição. Form permite colocar um input fora do formulário. Name caracteriza o input. Required campo deve ser preenchido antes de submeter. Placeholder deixa um valor escrito no campo (não trocar o label por esse atributo por conta de acessibilidade).

Password

Colocar senhas no formulário de maneira segura. Validar dados via minlength, maxlength, size (tamanho do campo), pattern (usa expressões regulares para validar senhas complexas), title (aparecerá caso o texto não esteja conforme o estabelecido), placeholder, required, inputmode (abre o teclado específico no smatphone, ex: numeric), autocomplete (permite sugestão do navegador para senhas salvas).

Email

Ele valida se o valor digitado é um e-mail, mas não se o e-mail é válido. Aceita placeholder, readonly, disabled, value, required, multiple (permite inserir mais de 1 e-mail separados por vírgulas), pattern (ex: pattern=".+rocketseat\.com\.br" só aceita e-mail rocketseat), title, list (ligado a uma datalist com e-mails sugeridos) minlength, maxlength e size.

URL

Espera que o usuário digite uma URL. Aceita placeholder, readonly, disabled, value, required, minlength, maxlength, size, pattern (ex: pattern=".*\.com\.br.*"), title, list e spellcheck (true or false, para fazer verificação de ortografia).

File

Usuário pode escolher 1 ou mais arquivos para carregar no fomrulário. Atributos value (arquivo a ser enviado), accept (tipos de arquivos aceitos, ex: .doc, .pdf ou image/*, image/png), files (lista de arquivos), multiple (permite envio de multipos arquivos). Quando inserido no formulário precisamos colocar o atributo enctype na tag form com o valor multipart/form-data.

Color

Interface para selecionar cor (color picker). Envia o valor em RGB. Aceita value e list. Não está disponível em alguns navegadores.

Checkbox

Caixas de seleção no formulário. Caso nada seja selecionado não será enviado nenhum valor. Recebe por padrão o valor on quando selecionado ou podemos personalizar pelo atributo value. Podemos também iniciar com ela selecionada pelo atributo checked. Além disso podemos colocar várias entradas ao usarmos o mesmo valor para name mas com values diferentes (ex: value=programação,musica).

Hidden

Invisível para o usuário e para leitores de tela. Usado para enviar algum valor necessário como por exemplo o timestamp (horário que foi enviado).

Radio

Selecionar uma única opção de um grupo de valores. Atributos essenciais são checked (indica se for marcado) e value.

Textarea

Colocar textos com mais de 1 linha, usado para comentários. Atributos: id, name, rows, cols, maxleng, minlength, wrap (padrão vem como soft, mas podemos desligar para pegar as quebras de linha somente do usuário com o texto excedendo a caixa), e outros comuns.

Select

Fornece um menu de opções através das tags option. Com o atributo multiple podemos selecionar várias opções com clique e arraste ou ctrl.

Option Group

Agrupa as opções dentro do select, não é selecionável e precisa do atributo label para indicar o grupo.

Search

Pra campos de busca, aparência semelhante ao texto. Atributos: list (para ajudar a procura), pattern (com title), aria-label (para leitores de tela).

Number

Entrada somente de números, aceita atributos de min, max, step(permite intervalo para selecionar os números) e os demais normais.

Range

Controle para selecionar um valor numérico do tipo slider ou dial. Atributos: min, max, value (valor inicial), step (passo do seletor).

Data e hora

Ostros inputs interessantes que não funcionam em todos os navegadores (sempre buscar no caniuse.com). Exemplos: date, datetime-local, month, time e week.

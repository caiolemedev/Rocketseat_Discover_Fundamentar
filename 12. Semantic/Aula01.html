O que é uma HTML semântica.

Web semântica

Adicionar significado ao conteúdo da página web, à estrutura. Existem convenções que são utilizadas para ajudar os useragents a identificar os componentes e apresenta-los corretamente, bem como ferramentas de busca e leitores de tela (acessibilidade).

Importância das tags HTML semânticas

Algumas tags possuem

significados específicos e orientações
claras sobre onde devem ficar na página e o motivo dela existir. Alguns elementos são utilizados em todos os sites, sempre buscando regras e padrões.

Seções comuns

Algumas delas são: cabeçalho (header), navegação (nav), conteúdo principla (main), barra lateral (aside) e rodapé (footer).

Header

Component estrutural e semântico do HTML. Quando utilizado dentro do body indica um cabeçalho global, caso seja utilizado dentro de uma section indicará o cabeçalho da section. Não é utilizado dentro do próprio header e no footer. Não possui atributos especiais e ocupa a linha toda da página.

Nav

Menu de navegação principal do site, normalmente em cima dentro do header ou na lateral. Podemos criar uma navegação específica em uma section, mas não é comum.

Main

Utilizar uma única vez por página dentro do body, inclui todo o conteúdo central relacionado à página. Não colocamos dentro de outras tags, somente no body.

Article

Construi um bloco de conteúdos relacionados, sobre o mesmo tema. Pode se repetir várias vezes na página e tem sentido sozinho fora da página.

Aside

Conteúdos extras, relacionados ao conteúdo principal. Explicações, propagandas, biografia, links extras, infos de perfil. Pode se repetir diversas vezes na página para cada aplicação.

Footer

Informações sobre o autor, dono da página, copyright, mapa da página. Usado de forma similar ao header.

Section

Constroi seções dentro do site e normalmente possui um título interno (h1, h2, etc.). Normalmente usado dentro de outras tags para criar um espaço, não é muito usada quando temos padrão de repetição dentro do site.

Elementos genéricos não semânticos

Aplicados para quando não temos uma tag semântica especifica e sempre usados com id e class para atribuir maior significado. Para bloco usamos div e para inline usamos span. Cuidado para não usar div em excesso.

O que podemos fazer com o NodeJS?

O NodeJS permite o desenvolvimento back e front end com JS, microserviços (aplicações completas), API (disponibilização de dados para o frontend), scripts, automação e até machining learning. Mas não é recomendado para grandes processamentos de CPU, exemplo: vídeos e fotos.

Vantagens

Muito rápido na execução e prototipagem, também tem uma excelente escalabilidade, usado no desenvolvimento de aplicações de ponta, usa o JS que é aplicado em vários lugares, Grade ecosistema e suporte. Temos casos famosos de uso: Netflix, Paypall, etc.

O que é o NodeJS?

O nodeJS é um JS Runtime Enviroment, ou seja, é um mini sistema operacional onde executa o JS internamente, mas consegue conversar com o sistema operacional quando necessário. Ele não é um framework (facilitador para a programação) ou uma linguagem de programação (não é, usa o próprio JS).

V8

Interpretador de JS para a linguagem da máquina criado em C++ e baseado nas ultimas features do JS, focado para o Chrome mas funciona no NodeJS. No node não temos a DOM e console, mas temos o file system. Node é composto pelo V8, Core Modules, C++ Bindings, libuv, c-ares, http, etc...

Como funciona?

Analogia dos restaurantes. Apache: 1 garçom pra cada cliente, ele fica bloqueado na cozinha até o pedido ficar pronto e depois de atender o cliente o garçom é despedido. Noderson: garçom Eveneto Lupin que trabalha de maneira assincrona e pode fazer mais de 1 evento por vez, sendo muito mais eficiente. Garçom = Thread, Pedido = Request, Chef = Banco de dados, Eveneto Lupin = Event Loop. O Node é single thread, non-blocking, asynchronous.

Avançando no entendimento e funcionamento

Criamos um aplicação em JS que se conversa com o V8 e o NodeJS, este se comunica com a libuv onde é rodado o event loop. Eventos que podem bloquear a aplicação são enviados para os worker threads (file system, networking, etc...) para serem executados, depois de completados o event loop recebem de volta o resultado via call back. O event loop é executado constantemente fazendo o leva e traz das informações. No node podemos registrar todas as funções e depois executar, como no JS e todo retorno vazio enviará um undefined. Podemos exemplificar a assincronia adicionando um setTimeout na aplicação que envia um retorno posterior, e mesmo com o time zero ele sai e depois retorna para a fila, portanto fica no final também.

De onde veio e para onde vai o NodeJS?

Foi criado em 2009 por Ryan Dahl e em 2011 foi liberado a NPM versão 1 que é o compartilhamento de bibliotecas e revolucionou o compartilhamento de códigos. Em 2015 foi criado o Node.js Foundation e passou a fazer parte da openjs Foundation. Comunidade muito ativa!

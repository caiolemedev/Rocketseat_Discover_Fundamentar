Events

Event Module

É um mecanismo para disparar eventos na aplicação, podemos também ouvir e realizar alguma ação ao identificarmos este evento. É a base para muitos outros módulos como: http, stream, file system, etc. Vamos aprender a utilizar, disparar, ouvir, executar ações com eventos.

Event Emitter

Podemos criar eventos via EventEmitter e salvar seus outputs em uma variável como no exemplo.

Emitindo eventos

Usamos a função emit na variável criada anteriormente para emitirmos um evento. Mas precisamos ouvir o mesmo, se não isso não adianta de nada, fica perdido no código.

Ouvindo e executando ações para os eventos

Usamos o on para ligar o ouvido da função e assim passamos a função (com ou sem inputs) que será executada sempre que o evento for acionado. No caso de inputs colocamos o mesmo logo após o nome do evento no emit e como parâmetro da função dentro do listener (on). Podemos executar a função quantas vezes desejarmos, basta somente emitir o evento com o nome determinado pois foi estabelecido um link com este.

Ouvindo eventos uma única vez

Se quisermos um event listener que escute a função uma única vez usamos o once no lugar do on, assim o primeiro evento será executado e os demais não.

Herdando do EventEmitter

Usando a const inherits podemos passar as funções do EventEmitter como no exemplo para um nova função que estamos criando em nossa aplicação. Assim, podemos executar as propriedades como manipulação de eventos no próprio objeto que estamos trabalhando.

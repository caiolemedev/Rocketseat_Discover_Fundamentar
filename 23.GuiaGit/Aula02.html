O que é Git

Controle de versão

Registro das alterações dos arquivos, reverter para estados anteriores, comparar mudanças, verificar quem modificou o arquivo, recuperação de projeto. Version Control System (VCS).

Tipos de controle de versão

Sistemas locais: a própria máquina, organização por pastas, duplicidade de arquivos, sem padrão, propenso a erros. Existe o RCS - Revision Control System que foi criado para evitar os erros anteriores, mas é um uso local e não permite o compartilhamento fácil e trabalho em equipe. Sistemas Centralizados: fica em um único servidor que são acessados por todos, controle de acessos e atividades dos colaboradores, fácil administração, se houver falha no servidor, corromper disco rígido e etc o projeto fica vulnerável. Sistemas Distribuídos: faz a duplicidade do repositório remoto localmente, cada pessoa trabalha independente dos demais, o clone é um backup completo, clientes usam o estado mais moderno.

O que é o Git

É um sistema de controle versão distribuído opensource. Ele salva os pontos na história do projeto e permite voltarmos em qualquer um desses pontos.

Usando Git em um projeto real

Iniciando um novo projeto

Baixar o arquivo da NLW e criar uma cópia no pc local. Usar o git initi para fazer com que ele se torne um repositório. Não podemos usar git init em pastas dentro de repositórios "pai", ele não criará um repositório interno.

Adicionando e verificando modificações

Alteramos o arquivo desejado usando o VS code, usamos o git diff para verificar qual linha foi alterada.

Staging e commits com atalho

Quando já temos o rastreamento dos arquivos modificados e não temos arquivos novos podemos pular a etapa do add e partir diretamente para o git commit -a ou -am "com menssagem".

Ver modificações em diversos pontos da história

Quando não temos modificações no working directory e no stage area não conseguimos usar o git diff para isso. No caso comparamos via git show IniciaisDoCommitAqui o git mostra todas as modificações feitas naquele commit. Podemos selecionar uma pasta ou um arquivo específico para vermos as modificações via git show IdDoCommit -- pasta/subpasta/NomeDoArquivo.

Ignorando arquivos e diretórios indesejados

Dentro do arquivo .gitignore podemos verificar quais arquivos estão sendo ignorados no repositório. Para pastas colocamos NomeDaPasta/ e para arquivos colocamos o NomeDoAquivo.

Conceitos

Estágios dos arquivos

Temos 3 estágios dos arquivos. Iniciando com git init para iniciar seu projeto ou git clone para copiar um projeto já existente. Estes arquivos estão sendo utilizados e transferimos para a stage area via git add e depois salvamos no repositório local via git commit. Modified -> Staged -> Commited.

Git Workflow

Segue os passos acima até salvar o no repositório local via git commit. O git salva todas as versões na linha do tempo.

Hash SHA-1

Converte os dados da alteração no formato SHA-1 para garantir a integridade dos dados e é um snapshot do programa. A partir do primeiro todos os commits são gerados a partir do pai, então é praticamente impossível ter dois ids de commit iguais.

HEAD

HEAD é uma seta que aponta para qual linha do tempo estamos alterando. Serve para trackearmos nosso projeto dentro da linha do tempo.

Revisão prática

Mostrando como iniciar um projeto, adicionar arquivos para a area de stage e depois commitar. No git status podemos verificar todos os detalhes do projeto, inclusive os arquivos não rastreados. Podemos adicionar ou remover arquivos da stage area antes do commit. Para remover da stage area usamos git rm --cached NomeDoArquivo.

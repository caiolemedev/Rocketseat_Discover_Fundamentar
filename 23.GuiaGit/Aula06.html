Alterando arquivos

Adicionando arquivos com git add

Revisão geral sobre git. Usamos git add NomeDoArquivo ou git add . , na primeira forma adicionamos arquivos individualmente e no segundo caso adicionamos todos os arquivos modificados de uma única vez. Podemos adicionar por extensão usando git add *.md ou *.html, etc.

Editando arquivos

Usamos o vim para modificarmos os arquivos e com git status podemos ver todas as modificações feitas. Adicionamos a stage area com git add e depois comitamos normalmente, o próprio git modificará os arquivos e controla-rá as versões do projeto.

Moficações com git diff

O comando git diff mostra todas as alterações de todos os arquivos, os -- indicam deleção de linha e arquivos, e o ++ significam que linhas e arquivos foram adicionados. Ele mostra também arquivos não ratreados, mas não consegue mostrar o comparativo.

Git diff staged

Pomdemos comparar os arquivos de duas maneiras. Via git diff - verificar working directory vs repository ou com git diff staged - compara staged com repository.

Deletando arquivos

Podemos deletar manualmente e depois inserir a remoção no stage area ou via git rm NomeDoArquivo diretamente. Após organizarmos a stage area podemos fazer o commit.

Renomeando arquivos

Podemos renomear na mão e depois adicionar ao stage area ou modificar o nome diretamente via git mv NomeAtual NomeNovo. Depois fazemos o commit.

Movendo arquivos

POdemos mover os arquivos manualmente ou via git mv NomeDoArquivo NomeDaPasta/NomeDoArquivo.

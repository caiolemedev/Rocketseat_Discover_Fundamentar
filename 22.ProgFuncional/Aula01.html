Por que programação funcional?

Organizar a linha de racioncínio para resolver os problemas, nova maneira de solucioná-los e é requisito para algumas tecnologias.

O que é Programação Funcional?

É um paradigma da programação, uma maneira de resolver os problemas, da mesma forma que a orientação a objetos. Mas esta se relaciona com funções, pequenas funções específicas para resolver o problema maior, cada uma modifica dados dentro do seu escopo. Temos um dado de entrada e a saída é um dado modificado, a função não guarda estados. Pontos positivos: fácil manutenção, fácil de testar e códigos mais confiáveis.

Princípios

Paradigmas: Programação Imperaviva vs Declarativas. Dados: Imutabilidade e Stateless. Funções: Independentes, Puras, Higher-order, First-class e Composição.

Programação Imperativa vs Declarativa

Imperativa cria funções que determinam como será feito, com passo a passo e determinadas, uma coisa depende da outra, POO é imperativo. Já a Declarativa cria função de o que será feito e não executa nada, só quando solicitada, funções mais genéricas, Prog Func é declarativa.

Imutabilidade

Uma variável que não será alterada, será uma constante. Em vez de alterarmos o objeto atual, criamos um novo objeto.

Stateless

Estados não são guardados, a função só conhece os dados que são estregues a ela, não alterando suas respostas.

Características das Funções

Tipos de funções

Funções Independentes

Deverá ter ao menos 1 argumento (dados informados) e não alterar dados externos, retorna algo obrigatóriamente, não guarda estado e não usa loops como for e while, quando necessário, usamos a recursividade.

Funções Puras

Não depende de nenhum dado externo, não sofre interferência de alterações externas, se o argumento é o mesmo o retorno é o mesmo, não tem efeito colateral, ou seja, não afeta o exterior.

Estrutura de Dados

Objetivo

O curso vai apresentar como são modeladas as estruturas, os conceitos fundamentais, tudo isso usando a linguagem JS. Uso dessa linguagem por ser a da Web e ser alto nível, foco no entendimento e não na construção.

Conhecimentos prévios

Intrudução a Web e o básico de programação, papel e caneta, VS Code, Navegador e NodeJS.

Por que estudar estrutura de dados?

Organizar os dados da aplicação, entender as estruturas para melhorar a tomada de decisão, melhorar os algorítimos, aumentar a eficiência, etc. Desafios diários: entendimento de como está o dado, criar listas, repetição dos dados, organizar os dados pelo par identificador + valor.

O que é Estrutura de Dados?

Uma maneira de organizar e ordenar informações, textos, números, booleanos, etc e registrá-los na memória do computador.

Gerenciamento de dados

É a gestão das informações da aplicação, podemos dividir em 3 etapas: modelar a estrutura, instanciar a estrutura (dar vida) e criar funcionalidades dessa estrutura (incluir, excluir, buscar, exibir, etc.).

Array

Também chamada de vetor, é uma estrutura amplamente utilizada e implementada em quase todas as linguagens de programação. É uma das entruturas mais básicas e simples de criar. O index, que são os espaços, é iniciado no 0 e ele respeita a ordem de inserção dos elementos. O array aceita valores duplicados e dependendo de seu tamanho precisaremos de uma performance maior. No JS os arrays são dinâmicos e podem aumentar ou diminuir conforme editamos. Pedemos ter dados misturados (object, string, number, etc.) dentro do mesmo array. Temos várias funcionalidades já implementadas como push(), pop(), find() e filter().

Array no código

Usando o vetores01 para testar algumas funcionalidade do array. Os detalhes estão no arquivo js.

Matrix

A matrix é um vetor multidimensional, ou seja, um array dentro de outro array.

Stack

Stack é uma pilha de algo, como se fosse uma pilha de livros. Ele é linear, um após o outro, então o último a entrar na pilha é o primeiro a sair (conceito last in, first out).

Stack no código

Alguns métodos fundamentais são: push(), pop() e pick(), sempre tratando o elemento do topo da stack. Podemos implementar outros métodos como size() para verificar o tamanho da pilha. Detalhes no arquivo stack01.

Queue

O conceito é de uma fila, usa o conceito de first in, first out onde a primeira a entrar é a primeira a sair. Enqueue = entrar na fila, Dequeue = sair da fila.

Queue

Implementando a fila no código adicionando os métodos enqueue e dequeue. Outros métodos podem ser implementados como size() para tamanho e front() para verificar o primeiro elemento da fila. Mais detalhes no arquivo queue em js.

Visão Geral

Usamos estrutura de dados para melhorar a organização e aumentar a eficiência em nossas aplicações. Existem muitas estruturas e cada uma é mais adequada a uma aplicação.

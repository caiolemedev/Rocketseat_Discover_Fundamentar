Selectors and Combinators

Introdução

Veremos como funcionam os seletores, combinadores, pseudo-classes e pseudo-elements

Seletores

Conecta um elemento HTML ao CSS para podermos alterar o elemento. Podemos usar o id como seletor, mas devemos ter um id único para cada página, usa o # antes do nome no css. Para classe podemos ter mais de um elemento com a mesma classe na página, o css irá alterar todos os elementos da classe, usamos . antes do nome no css. Seletor por atributo funciona similar a classe mas colocamos o nome no css entre chaves. Pedemos usar pseudo-classes que terão um estado específico. Podemos agrupar formatações do css via seleção de múltiplos elementos separados por vírgula.

Combinators

Buscam e combinam seletores para aplicar uma estilização. Mesmo que haja elementos no meio do caminho todos os elementos dentro do pai sofrerão as alterações.

Child combinator

Ele seleciona somente elementos que são filhos diretos do pai quando usamos o sinal de maior >.

Adjacent sibling combinator

Identificado pelo sinal de mais + entre dois seletores. Seleciona somente o elemento do lado direito que é irmão direto na hierarquia. Normalmente usado para colocar espaço entre os botões.

General sibling combinator

Identificado pelo ~ entre os elementos e considera todas as tags irmãs dentro e não só o primeiro como no anterior.

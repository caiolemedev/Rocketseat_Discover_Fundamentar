Trabalhando com repositórios

Criando chave SSH

Precisamos fazer esse mesmo processo em todas as máquinas que queremos ligar com o repositório do GitHub.Colocar no terminal: ssh-keygen -t rsa -b 4096 emaildogithub@email.com, seguimos dando enter até o terminal apresentar a chave, para o windows usamos o terminal do git. Para pegar a chave usamos a o comando: cd ~/.ssh/id_rsa e depois um cat ~/.ssh/id_rsa.pub e copiar o conteúdo. Após isso no GitHub inserimos a chave e colocamos uma descrição do computador.

Adicionando chaves ao gerenciador local SSH Agent

Precisamos executar uma etapa adicional para funcionar certo as chaves SSH do GitHub com o pc.

Linkando um repositório remoto com local

Criar um repositório online no próprio GitHub ou via linha de comando (no GitHub tem um manual de como fazer). Via command line usar a chave SSH para ligar os dois repositórios, em vez de HTTPS. Dúvidas acessar o GitHub do maykbrito/demo. Para criar branch usamos o upstream.

Modificando arquivos local e enviando para repositório remoto

Após alterarmos o arquivo, seguimos a sequência git add e git commit para adicionarmos ao repositório local. Como já foi configurado anteriormente, com o git push atualizamos os dois repositórios (local e remoto) de uma única vez.

Modificando arquivos remotos e puxando para o repositório local

Editamos o arquivo e clicamos em commit change. O repositório local fica desatualizado, para isso usamos git pull e o git atualizará o repositório local.

Entendendo o histórico remoto

Podemos entrar nos commits do projeto e verificar todas as modificações linha a linha ou verificar o código no momento do commit.

Corrigindo conflitos de merge

Quando temos conflito de código (editamos local e remoto em overlap) precisamos resolver os conflitos no código via merge para que o git e o GitHub possam atualizar todos os repositórios corretamente. Abrimos o repositório no VS code e podemos aceitar ou rejeitar fácilmente.

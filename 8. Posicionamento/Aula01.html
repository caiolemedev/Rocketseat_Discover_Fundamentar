Posicionamento e Layout

Layout

Antes eram feitos em tabela, mas depois surgiram os conceitos de frameworks e grid systems e finalmente flexbox e grid que são bem melhores.

Position

O fluxo dos elementos é um abaixo do outro (tipo block) ou um ao lado do outro (inline). É possível utilizar mais de uma classe no elemento.

Static

Os elementos seguirão o fluxo normal da página.

Relative

A propriedade position relative ela abre novas propriedades de posicionamento. Mantem os demais elementos no fluxo normal.

Absolute

Também abre as novas propriedades de posição mas altera o fluxo da página pois insere uma nova camada para o elemento. Caso haja um elemento pai com posicionamento relativo ele fará referência a este elemento.

Fixed

Usando o position fixed ele se mantem fixo na página, não sendo afetado pela rolagem.

Element Stacking

É a propriedade de z index que empilha os elementos na sequência que aparecem na página, mas é possível alterar via propriedade z-index.

Flex

Permite posicionar os elementos dentro da caixa, normalmente utizamos um elemento container e podemos alterar os posicionamentos dos elementos dentro dele.

Grid

Posicionamento de elementos dentro da caixa também, com posicionamentos horizontais e verticais ao mesmo tempo. Cria espaço para os elementos filhos como uma tabela e permite configurar as linhas e colunas.

Grid e Flex juntos

Podemos usar o flex e o grid juntos, desde que não seja no mesmo elemento, por exemplo um flex dentro do elemento header para posicionar uma logo e um menu.
